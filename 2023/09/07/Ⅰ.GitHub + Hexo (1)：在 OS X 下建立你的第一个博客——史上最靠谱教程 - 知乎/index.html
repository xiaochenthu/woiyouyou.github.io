Ⅰ：GitHub + Hexo (1)：在 OS X 下建立你的第一个博客——史上最靠谱教程 - 知乎前言建立个人博客的方法很多， GitHub + Hexo 只是其中的一种，但这种方法几乎不需要基础，上手简单，维护方便，是建立个人博客的理想方法。关于系列教程，我推荐的阅读顺序为：GitHub + Hexo (1)：在 OS X 下建立你的第一个博客——史上最靠谱教程GitHub + Hexo (2)：部署博客及更新博文GitHub + Hexo (3)：安装主题GitHub + Hexo (4)：配置第三方服务GitHub + Hexo (5)：总结
－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－这是我真正意义上的第一篇文章，起因是21个小时前看了一篇win教程“ 用 GitHub + Hexo 建立你的第一个博客”，刚好又有些空余时间，于是实践了下。没想到这次实践困难重重，加之网上所能搜索到的教程残缺不全，因而自己动手写下这篇教程。本文适合小白看，完全白纸的都可以，也欢迎大神们不吝赐教。关于GitHub和Hexo的介绍以及GitHub注册的步骤我就不赘述了。

安装软件一、打开Hexo官网1、官网是这个样子的，打开速度可能会比较慢，耐心等一等。
2、右上角将语言切换为简体中文（最初我并没有注意到语切换，直接就英文看完了全程，摔），随后点击中部的箭头，进入简略教程。
3、页面中所提示的这部分我没有遇到。Mac 用户您在编译时可能会遇到问题，请先到 App Store 安装 Xcode，Xcode 完成后，启动并进入 Preferences -> Download -> Command Line Tools -> Install 安装命令行工具。
二、安装Git1、这里直接点击蓝色字体的“安装程序”，随后的界面将是这样的。2、点击“download”，随后在你的下载目录中找到.dmg文件，双击安装。三、安装 Node.js1、首先打开Terminal，如果不知道在哪儿的话直接全局搜索，中文应该叫“终端”，长这样子。2、输入$ curl https://raw.github.com/creationix/nvm/master/install.sh | sh这里实际上不需要cd到一个新的文件夹，直接安装就好。3、到这里还是一切正常，随后麻烦就来了。我们重启Terminal（右击Dock中的图标，选择Quit，然后再打开）并输入$ nvm install stable得到的将是这个，反复几次都是如此。4、官网并没有对这个现象做出详细的解释，在这里学长给出解答。这是因为没有配置环境变量，比如我的电脑里就只有python 3.5的环境变量。在Terminal中输入$ vim ~/.bash_profile就可以查看当前所有的环境变量。5、解决的方案在这里——安装nvm后，运行nvm,输出-bash:nvm:command not find输入$ curl -o-https://raw.githubusercontent.com/creationix/nvm/v0.25.4/install.sh | bash
6、然后我们重复步骤3，将得到下图，到这里Node.js就安装成功了。四、安装 Hexo1、在Terminal中输入，在这里我碰到第二个问题。$ npm install -g hexo-cli2、这是由于npm ERR! Please try running this command again as root/Administrator.因而改进代码，输入$ sudo npm install -g hexo-cli3、此时会要求输入Password，这里应该是你电脑的开机密码，注意光标是不会移动的，正常输入密码就好。4、安装完成时的界面是这样的。建站1、安装 Hexo 完成后，输入$ hexo init Hexo将会在指定文件夹中新建所需要的文件。注意你自己新建文件夹的时候是不要“<>”的。
2、完成时是这样的（其实这里有问题）。3、随后输入$ cd
$ npm install
然后问题就来了，Terminal没反应，包括随后的几个步骤都是得不到正确回应的。
4、和上面的问题一样，改进代码，得到5、完成时应该是这样。本地测试1、cd到刚才创建的文件夹，否则将会是这样。2、cd到刚才创建的文件夹，例如我就是$ cd hello回车后输入$ hexo g3、再输入$ hexo s4、查看成果的时候到了，打开Terminal下方的网址，大功告成。当然，这步可能会非常慢，我等了好久，以至于以为又出了什么bug。5、最后关闭测试。
