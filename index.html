III：GitHub + Hexo (3)：安装主题-这篇文章将以NexT为例，介绍“安装主题”的相关步骤
－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－－
我们之前使用 Hexo 生成的博客，默认主题是 Landscape ——显然这个主题不够酷炫，而搭建个人博客中非常主要的一个步骤就是选择一个酷酷的主题并配置好。

选择主题你可以直接浏览 hexo 主题汇总，从这中间挑选一个中意的主题，当然也可以参考大神的推荐

我这里所做的范例就是高票推荐中的最高票主题NexT。
NexT“精于心，简于形”的标语非常合我的口味，视觉上实际有一点点日系风格，当然这是我个人的看法，你也可以选择其他主题，主题配置大部分还是大同小异的。

安装主题

3.1背景说明在 Hexo 中有两份主要的配置文件，其名称都是 _config.yml ——一份位于站点根目录下，主要包含 Hexo 本身的配置；另一份位于主题目录（themes）下，这份配置由主题作者提供，主要用于配置主题相关的选项。
为了描述方便，在以下说明中，将前者称为站点配置文件， 后者称为主题配置文件。

3.2下载主题先 cd 到 Hexo 站点目录下，例如我的是 Hexo$ cd Hexo
$ git clone https://github.com/iissnan/hexo-theme-next themes/next

3.3启用主题打开站点配置文件， 找到 theme 字段，并将其值更改为 next 。

3.4验证主题开启调试模式（即加上 —debug），输入：$ hexo s —debug不出意外将看到如下的反馈：INFO Hexo is running at http://0.0.0.0:4000/. Press Ctrl+C to stop.这时候在浏览器中打开http://0.0.0.0:4000/就可以看到你的博客了，但是这时候的博客还是默认的主题，我们需要对主题进行配置。

主题设定

4.1选择 SchemeScheme 是 NexT 提供的一种特性，用于提供不同的外观。目前 NexT 支持三种 Scheme：
· Muse - 默认 Scheme，这是 NexT 最初的版本，黑白主调，大量留白
· Mist - Muse 的紧凑版本，整洁有序的单栏外观
· Pisces - 双栏 Scheme，小家碧玉似的清新
Scheme 的切换通过更改 主题配置文件，将需用启用的 scheme 前面注释“ #” 去掉即可，例如我现在启用的是 Mist 。

4.2设置语言编辑 站点配置文件， 将 language 设置成你所需要的语言。我选择的是英文，配置如下：language: en

4.3设置站点名称编辑站点配置文件， 设置 title 字段为你的站点名称：

4.4设置菜单菜单配置包括两个部分，第一是菜单项（名称和链接），第二是菜单项的显示文本，第三是菜单项对应的图标。 编辑主题配置文件 ,修改以下内容：
设定菜单内容对应的字段是 menu，格式为item name: link。其中 item name 是一个名称，并不直接显示在页面上，而是用于匹配图标以及翻译，例如我的设置为：
menu:
home: /
#categories: /categories
about: /about
archives: /archives
#tags: /tags
#commonweal: /404.html
NexT 默认的菜单项有（除前两项外均需要手动创建这个页面）：

4.5设置菜单项的显示文本在第一步中设置的菜单的名称并不直接用于界面上的展示。Hexo 在生成的时候将使用这个名称查找对应的语言翻译，并提取显示文本。这些翻译文本放置在 NexT 主题目录下的 languages/{language}.yml （{language} 为你所使用的语言）。由于我所用的是英文，因而这一步就略去了。

设定菜单项的图标对应的字段是 menu_icons。 此设定格式是 item name: icon name，其中 item name 与上一步所配置的菜单名字对应，icon name 是 Font Awesome 图标的 名字。而 enable 可用于控制是否显示图标，你可以设置成 false 来去掉图标。对图标我还是不甚了解，这点容我后面再摸索下。

设置头像编辑站点配置文件， 新增字段 avatar， 值设置成头像的链接地址。我推荐使用“完整的互联网 URI”，这是最方便快捷的。

4.6设置作者昵称编辑站点配置文件， 设置 author 为你的昵称：author: xxx

4.7站点描述编辑站点配置文件， 设置 description 字段为你的站点描述：description: xxx
